Задание будет выполняться постепенно и возможно с некими поправками(будут описаны ниже).


уровень сложности: высокий.

Время исполнения: 8 часов.

Ограничения по технологиям: Стандарт языка 11|14|17. Фреймворки и библиотеки со свободной лицензией. (без библиотеки curl и её производных)

Ограничения памяти: Размеры загружаемых файлов не должны влиять на общее потребление памяти программы.

Вид программы: консольное приложение.

Система сборки: Cmake.

Способ передачи программы: публикация исходного кода на github или gitlab по выбору кандидата.

Дополнительные требования: Программа должна успешно компилироваться в linux с использованием копилятора Gcc (версия соответствует выбранному стандарту языка), в windows с использованием компилятора MSVC версии, соответствующей выбранному стандарту языка.



Входные параметры.

Программе в параметрах запуска передаётся две строки. (строки не содержат пробельных символов внутри себя) и число в деястичной форме. Параметры позиционные. Присутствуют всегда.

Первая содержит путь (абсолютный либо относительный) до файла со списком URL. Гарантируется что путь корректный, файл существует, программа обладает правами на чтение файла.

Вторая строка содержит путь до каталога, в который необходимо разместить загруженные файлы. Гарантируется, что путь является валидным для целевой системы и у программы есть все необходимые права на создание необходимых каталогов и запись файлов. Указанный каталог и путь к нему могут не существовать.

Третий параметр - положительное целове число представленное в десятичной форме в диапазоне [1..999]

Файл с URL.

Во входном файле в кодировке utf-8 содержатся URLы по одному на строку. Пример:

https://bolid.ru/favicon.ico
https://bolid.ru/bld/images/logo.png
Разделителями строки являются стандартные CR LF либо LF. В строках не содержится лишних пробельных символов. Разделитель в конце последней строки (пустая строка в конце файла) может присутствовать.

Допустимы адреса с http и https. Для https валидации сертификата не требуется.

Требуется сохранить на диск файлы по заданным URL.

Именем сохраняемого файла берётся имя из заголовка "Content-Disposition" типа "attachment". В случае отсутствия заголовка именем файла берётся часть строки изначального URL после последнего символа "/". Невалидные для имени символы необходимо заменить символом "_".

В случае если файл с таким именем уже существует - к имени файла добавляется номер (1), (2), и так далее в случае необходимости.

В случае если код ответа от сервера отличается от 200 ОК - код ошибки выводится в консоль, URL игнорируется. (Обработка редиректов не требуется).

Загрузка каждого файла должна выполнятся в отдельном соединении (оптимизация с keep-alive не требуется).

Одновременно должна вестись загрузка количества файлов, указанного в третьем параметре, переданном программе. Способ распараллеливания загрузки на усмотрение кандидата.



Вывод на консоль.

Программа должна фиксировать в стандартный вывод следующие события: запуска программы, входные параметры, начало загрузки файла, окончание загрузки файла, ответ от сервера с кодом отличным от 200 ОК, завершение программы.

Вывод допустим в произвольной форме. Обязательно должен содержать время с точностью не ниже миллисекунд.



Оформление программы.

Сборка программы должна осуществляться системой Cmake.

Программа должна быть размещена на github или gitlab.

В корне репозитория должен содержаться README.md с описанием программы а так же указанием списка требуемых для сборки библиотек (включая версии) со ссылками на эти библиотеки (репозитории github\gitlab, страницы загрузки на официальных сайтах).

Юнит тесты приветствуются.
